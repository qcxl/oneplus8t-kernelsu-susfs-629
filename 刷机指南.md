## 准备工作

### 你需要的东西

| 项目 | 说明 |
|------|------|
| ✅ 一台电脑（Windows / macOS / Linux 均可） | 用于执行刷机命令 |
| ✅ 一根 USB 数据线 | 连接手机和电脑 |
| ✅ OnePlus 8T 手机 | 已解锁 Bootloader（不解锁不能刷） |
| ✅ 手机电量 > 50% | 防止刷机过程中断电 |

### 什么是解锁 Bootloader？

手机出厂时 Bootloader 是锁定的，不能刷第三方内核。OnePlus 8T 解锁方法：

1. 手机进入 "设置 → 关于手机 → 版本号"，连续点击 7 次 "版本号" 开启开发者模式
2. 回到 "设置 → 系统 → 开发者选项"，开启 **OEM 解锁**
3. 关机后同时按住 **音量上 + 音量下 + 电源键** 进入 fastboot 模式
4. 连接电脑，运行：`fastboot oem unlock`（会清除所有数据！）

> ⚠️ 解锁会清除手机全部数据，请先备份照片等重要文件。

## 第一步：下载需要的文件

### 1.1 下载 boot.img（内核镜像）

打开浏览器，访问 GitHub Actions 页面：

```
https://github.com/qcxl/oneplus8t-kernelsu-susfs-629/actions
```

找到最新成功的构建（绿色 ✅），点进去，在底部 **Artifacts** 部分下载 `kebab-kernel-ksu-debug.zip`。

解压 zip 文件，里面有一个 `ksu-debug-boot.img`，这就是我们要刷的文件。

> 💡 **如何判断构建成功**：在 Actions 列表中找带有绿色 ✅ 标记的最新一条，点开确认所有步骤都打绿勾。

### 1.2 下载 KernelSU-Next App

在同一个 CI 构建页面，找到名为 `ksu-manager` 的 Artifact，下载 `KernelSU_Next_*.apk` 文件。

或者从 KernelSU-Next 的 CI 下载：

```
https://github.com/rifsxd/KernelSU-Next/actions
```

找到最新 dev 分支的成功构建，下载 `ksu-manager` 中的 APK。

### 1.3 下载 ksud（可选，用于修复 App SELinux 开关）

如果 build-kernelsu-susfs 的 CI 有 `ksu-ksud` artifact，也一并下载。里面有一个 `ksud` 文件（没有后缀名）。

> 注意：ksud 是 Linux 可执行文件，不是 Windows 程序。它会被推送到手机上运行。

## 第二步：安装刷机工具（电脑上操作）

### 2.1 安装 ADB 和 Fastboot

ADB（Android Debug Bridge）和 Fastboot 是刷机必需的工具。

**Windows 用户**：
1. 下载 [Platform Tools](https://developer.android.com/studio/releases/platform-tools)
2. 解压到 `C:\platform-tools`
3. 打开命令提示符（按 Win+R，输入 `cmd`，回车）
4. 输入：`cd C:\platform-tools`
5. 以后所有命令都在这个窗口执行

**macOS 用户**：
```bash
# 用 Homebrew 安装（如果没有 Homebrew，先去 https://brew.sh 安装）
brew install android-platform-tools
```

**Linux 用户**：
```bash
sudo apt install android-tools-adb android-tools-fastboot
```

验证安装成功：
```bash
adb --version
fastboot --version
```

### 2.2 准备文件

把下载的 `ksu-debug-boot.img`、APK 文件、`ksud` 文件全部放到电脑同一个文件夹里（比如 `C:\刷机` 或 `~/刷机`）。

## 第三步：手机进入 fastboot 模式

### 方法一（推荐）：通过 ADB 命令重启

手机正常开机，连接电脑，打开命令行/终端执行：
```bash
adb reboot bootloader
```

### 方法二：手动按键

1. 关机
2. 同时按住 **音量上 + 音量下 + 电源键** 约 5 秒
3. 手机屏幕显示 "FASTBOOT" 字样即成功

验证手机已进入 fastboot：
```bash
fastboot devices
```
应该显示一行类似 `e4a9c6b1 fastboot` 的输出。

## 第四步：刷入临时内核

> ⚠️ **重要**：我们使用 `fastboot boot` 命令，**不是** `fastboot flash`。区别：
> - `fastboot boot` = 临时启动新内核，重启手机后恢复原样 ❌ 不会变砖
> - `fastboot flash` = 永久写入内核，重启也还在 ❗ 有风险

首先确认 boot.img 文件路径，假设文件在 `C:\刷机\ksu-debug-boot.img`：

```bash
# Windows 用户
fastboot boot C:\刷机\ksu-debug-boot.img

# macOS / Linux 用户
fastboot boot ~/刷机/ksu-debug-boot.img
```

看到以下输出说明成功：
```
Sending 'boot.img' (98304 KB)    OKAY
Booting                            OKAY
```

手机重启，等待 1-2 分钟进入系统。

### 验证新内核是否生效

```bash
adb shell uname -r
```

应该显示：`4.19.304-perf-g5dea892fe7e4-dirty`

```bash
adb root
adb shell dmesg | grep "kernelsu_init"
```

应该看到：`kernelsu_init+0x0/0x15c returned 0`

这两条都对了就说明新内核正常运行。

## 第五步：安装 KernelSU-Next App

### 5.1 开启 USB 安装权限

手机连接电脑，下拉通知栏，把 USB 选项从 "仅充电" 改为 "文件传输" 或 "传输文件"。

手机上进入 "设置 → 安全 → 允许安装未知来源应用"。

### 5.2 安装 APK

```bash
# 先卸载旧版（如果有）
adb uninstall com.rifsxd.ksunext

# 安装新版
adb install 路径/KernelSU_Next_xxx.apk
```

看到 `Success` 说明安装成功。

### 5.3 授予权限

打开手机上的 KernelSU-Next App：
- 允许通知权限（按提示点"允许"）
- 打开 "设置 → 超级用户" 页面，确认能正常显示

## 第六步：更新 ksud（让 App 的 SELinux 开关生效）

> 这一步是可选但建议的。如果不做，App 的 "SELinux 宽容模式" 开关可能无法正常工作。

```bash
# 把 ksud 文件推送到手机
adb push 路径/ksud /data/local/tmp/

# 复制到 ksud 的正确位置并设置权限
adb shell su -c "cp /data/local/tmp/ksud /data/adb/ksu/bin/ksud"
adb shell su -c "chmod 755 /data/adb/ksu/bin/ksud"
adb shell su -c "chown root:root /data/adb/ksu/bin/ksud"

# 验证版本
adb shell /data/adb/ksu/bin/ksud debug version
```

## 第七步：测试所有功能

### 7.1 验证内核功能

```bash
# 检查是否 panic
adb shell dmesg | grep -i "panic\|BUG\|Oops"
# 应该没有任何输出（没有就是好消息）

# 检查所有功能是否注册成功
adb shell dmesg | grep "registered handler"
# 应该看到 id=0,1,2,3,4,5,10003 全部注册
```

### 7.2 测试 Hide SELinux（隐藏 SELinux 修改）

在 App 设置中：
1. 打开 "Hide SELinux modification"
2. 关闭后再打开
3. 手机应该正常工作，不会崩溃

或者用命令测试：
```bash
adb shell /data/adb/ksu/bin/ksud feature set 4 1
adb shell /data/adb/ksu/bin/ksud feature set 4 0
```

### 7.3 测试 SELinux 宽容模式切换

在 App 设置中：
1. 找到 "SELinux" 开关，点击切换
2. 如果提示 "已设置为宽容模式" 说明成功

或者用命令测试：
```bash
adb shell /data/adb/ksu/bin/ksud feature set 5 0   # 宽容模式
adb shell /data/adb/ksu/bin/ksud feature set 5 1   # 强制模式
```

### 7.4 测试 ADB Root

在 App 设置中测试 "ADB Root" 开关的切换。

### 7.5 测试 SU Compat（传统 SU 支持）

在 App 设置中测试 "传统SU命令支持" 开关。

> 以上所有功能切换都应该不引起手机崩溃。如果某个开关导致手机死机/重启，说明有 Bug。

## 第八步（可选）：永久刷入内核

> **建议先用 `fastboot boot` 临时模式稳定运行几天**，确认没有问题后再永久刷入。

```bash
# 查看当前槽位（A/B 分区的手机有两个槽位）
adb shell getprop ro.boot.slot_suffix
# 常见结果：_a 或 _b

# 备份当前内核（以防万一）
adb root
adb shell "dd if=/dev/block/by-name/boot$(adb shell getprop ro.boot.slot_suffix) of=/data/local/tmp/stock-boot.img"
adb pull /data/local/tmp/stock-boot.img ~/stock-boot-backup.img

# 永久刷入新内核
adb reboot bootloader
fastboot flash boot_a 路径/ksu-debug-boot.img   # 如果槽位是 _a
# 或
fastboot flash boot_b 路径/ksu-debug-boot.img   # 如果槽位是 _b

# 重启手机
fastboot reboot
```

> ⚠️ 永久刷写有风险。如果内核有问题，手机可能无法开机。务必先备份原内核。

## 附：解决 WiFi 感叹号（网络连接受限）

### 问题现象

- WiFi 已连接，但状态栏有感叹号 ⚠️
- 显示"网络连接受限"
- 实际浏览器能打开网页

### 原因

Android 系统会检测能否访问 Google 服务器来判断网络是否正常。你的手机到某些 Google IP 的 HTTPS 连接被阻断，导致系统误判为"网络受限"。

### 解决方法

连接电脑，执行以下命令：

```bash
adb shell settings put global captive_portal_https_url http://connectivitycheck.gstatic.com/generate_204
```

然后关闭 WiFi 再打开，感叹号应该消失。

### 如果需要恢复默认设置

```bash
adb shell settings delete global captive_portal_https_url
```

## 附：救砖（如果刷坏了如何恢复）

### 情况 A：还能进入 fastboot 模式

如果你刷了有问题的内核，手机卡在开机画面或一直重启：

1. 强制重启到 fastboot：
   - 长按电源键 10 秒直到关机
   - 同时按住 **音量上 + 音量下 + 电源键** 进入 fastboot

2. 用之前备份的内核启动（前提是你备份了）：
   ```bash
   fastboot boot ~/stock-boot-backup.img
   ```

3. 或者永久刷回原内核：
   ```bash
   fastboot flash boot_a ~/stock-boot-backup.img
   fastboot reboot
   ```

### 情况 B：既不能开机也不能进 fastboot

这种情况很少见。尝试：
1. 长按电源键 30 秒以上强制关机
2. 同时按住 **音量上 + 音量下 + 电源键** 尝试进入 fastboot
3. 如果不行，搜索 "OnePlus 8T EDL mode" 进入深度刷机模式（需要特殊的工程线）

### 最佳预防

**每次刷机前一定要备份原内核：**
```bash
adb shell "dd if=/dev/block/by-name/boot$(adb shell getprop ro.boot.slot_suffix) of=/data/local/tmp/stock-boot.img"
adb pull /data/local/tmp/stock-boot.img /安全的地方/stock-boot-日期.img
```

## 常见问题

### Q：fastboot boot 卡在 "Sending" 不动？

- USB 线可能有问题，换一根线试试
- 换一个 USB 接口（台式机用机箱后面）
- 重启电脑和手机再试

### Q：刷完内核手机一直重启（bootloop）？

- 音量键按到停不下来？检查音量键是否卡住
- 强制重启到 fastboot，用备份内核恢复
- 试一下刷另一边的槽位：`fastboot flash boot_b xxx.img`

### Q：App 安装失败显示 "INSTALL_FAILED_UPDATE_INCOMPATIBLE"？

- 表示有旧版 App 冲突，先卸载再安装：
  ```bash
  adb uninstall com.rifsxd.ksunext
  adb install 路径/xxx.apk
  ```

### Q：ksud 命令提示 "not found"？

```bash
# ksud 的正确路径是
adb shell /data/adb/ksu/bin/ksud
```

### Q：App 的 SELinux 开关点了没反应？

- 说明 ksud 版本太旧，参考[第六步](#第六步更新-ksud让-app-的-selinux-开关生效)更新 ksud
- 或者直接通过 adb 命令切换：
  ```bash
  adb shell /data/adb/ksu/bin/ksud feature set 5 0   # 宽容模式
  adb shell /data/adb/ksu/bin/ksud feature set 5 1   # 强制模式
  ```

### Q：怎么知道内核是否正常工作？

```bash
# 查看内核版本
adb shell "uname -r"

# 查看是否有崩溃
adb shell "dmesg | grep -i 'panic'"

# 查看 KSU 功能是否注册
adb shell "dmesg | grep 'registered handler'"
```
